🛸 UFO Sightings Trend Forecasting
Project for Machine Learning Deployment Course

[]([ใส่ URL Streamlit ของคุณที่นี่])

1. การนิยามปัญหา (Problem Definition)
ปัญหา: ข้อมูลการพบเห็น UFO มักถูกมองว่าเป็นเรื่องบังเอิญหรือคาดเดาไม่ได้ แต่ในความเป็นจริง รายงานเหล่านี้อาจมีรูปแบบเชิงเวลาที่ซ่อนอยู่ การไม่สามารถคาดการณ์แนวโน้มการพบเห็นในอนาคตทำให้ยากต่อการแยกแยะระหว่าง "ช่วงเวลาปกติ" กับ "ช่วงที่มีรายงานสูงผิดปกติ" ซึ่งอาจสัมพันธ์กับกิจกรรมทางดาราศาสตร์หรืออากาศยาน

เป้าหมาย: สร้างโมเดล Machine Learning เพื่อพยากรณ์จำนวนการพบเห็น UFO ในอนาคต โดยวิเคราะห์จากข้อมูลสถิติย้อนหลัง เพื่อให้เข้าใจแนวโน้ม (Trend) และฤดูกาล (Seasonality) ของปรากฏการณ์นี้

ทำไมถึงใช้ ML: เนื่องจากข้อมูลมีปริมาณมหาศาล (80,000+ รายการ) และมีความผันผวนสูง การใช้สถิติพื้นฐานไม่สามารถจับรูปแบบที่ซับซ้อน เช่น การทับซ้อนของวงจรรายเดือนและรายปีได้ ML (โดยเฉพาะโมเดล Time Series) จึงเหมาะสมในการสกัด Pattern เหล่านี้ออกมา

2. ข้อมูลที่ใช้ (Dataset Information)
ที่มา: UFO Sightings (scrubbed.csv) https://www.kaggle.com/datasets/NUFORC/ufo-sightings?select=scrubbed.csv รวบรวมโดย National UFO Reporting Center (NUFORC)
ความน่าเชื่อถือ: เป็นฐานข้อมูลที่เก่าแก่และใหญ่ที่สุดชุดหนึ่งของโลก โดยฉบับ "Scrubbed" ได้ผ่านการทำความสะอาดพิกัดภูมิศาสตร์และจัดการข้อมูลที่ผิดพลาดเบื้องต้นมาแล้ว

ความหมายของ Features ในบริบทของโปรเจค:
datetime: ใช้เป็นแกนหลัก เพื่อวิเคราะห์ Trend และ Seasonality
country/state: ใช้ระบุขอบเขตพื้นที่เพื่อลดความคลาดเคลื่อนของข้อมูลจากปัจจัยท้องถิ่น
duration (seconds): ใช้กรองข้อมูลที่มีความยาวผิดปกติ (Outliers) เพื่อเพิ่มความแม่นยำให้โมเดล
shape: ลักษณะทางกายภาพของวัตถุที่ใช้ในการวิเคราะห์พฤติกรรมการรายงานตามยุคสมัย

3. กระบวนการพัฒนาโมเดล (Model Development)
เพื่อให้เป็นไปตามเกณฑ์การเปรียบเทียบโมเดลอย่างเป็นระบบ โปรเจคนี้ได้ทำการทดลองดังนี้:

Baseline Model: Moving Average (ค่าเฉลี่ยเคลื่อนที่) เพื่อดูแนวโน้มพื้นฐาน

Advanced Model: Facebook Prophet เนื่องจากรองรับการจัดการ Seasonality และวันหยุดได้ดีเยี่ยม

Optimization: ทำการปรับจูน Hyperparameters (เช่น changepoint_prior_scale) ผ่านกระบวนการทดลองเพื่อหาค่า Error (RMSE/MAE) ที่ต่ำที่สุด

🚀 4. การใช้งาน (Deployment)
ผู้ใช้สามารถเข้าใช้งานโมเดลผ่าน Web Application (Streamlit) โดยมีฟีเจอร์หลักดังนี้:
Dashboard: แสดง Insights จากการวิเคราะห์ข้อมูลย้อนหลัง (EDA)
Prediction: เลือกช่วงเวลาในอนาคตเพื่อพยากรณ์จำนวนการพบเห็น UFO
Reliability: ระบบมีการทำ Input Validation เพื่อป้องกันการใส่ข้อมูลที่ผิดพลาด
